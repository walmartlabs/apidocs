composite-type

(composite-type v)

Returns :map, :seq, :vec. This key is used when identifying empty values and so forth.

render-composite

(render-composite v state)

Renders the value as a new node (this is often quite recursive).

Returns a tuple of the new state, and the node-id just rendered.

State has a number of keys:

:values
map from a non-scalar value to its unique id; this is used to create edges, and track which values have already been rendered.
:nodes
map from value id to rendered text for the node; rendering adds a value to this map.
:edges
map from node id to node id; the source node is may be extended with a port (e.g., “map_1:v3”)"

use-identity?

(use-identity? v)

Returns true if it is not possible to cache the mapping of value to node id. This is true for (most) sequences, as sequences can be infinite and therefore make bad keys (it takes infinitely long to compute the key hash).

It is false for all finite types (map, vector, and so forth).
